  7   # Converts an array into a hash. Converting an array
  8   # into a hash is not a one-to-one conversion, for this
  9   # reason #to_h examines at the array being converted
 10   # and then dispatches the conversion to the most sutiable
 11   # specialized function. There are three possiblities for this.
 12   #
 13   # If the array is a collection of perfect pairs, like that
 14   # which Hash#to_a generates, then conversion is handled by 
 15   # #to_h_flat.
 16   #
 17   #   a = [ [:a,1], [:b,2] ]
 18   #   a.to_h  #=> { :a=>1, :b=>2 }
 19   #
 20   # If the array contains only arrays, but are not perfect pairs,
 21   # then #to_h_multi is called.
 22   #
 23   #   a = [ [:a,1,2], [:b,2], [:c], [:d] ]
 24   #   a.to_h  #=> { :a=>[1,2], :b=>[2], :c=>[], :d=>[] }
 25   #
 26   # If the array contians objects other then arrays then
 27   # the #to_h_splat method is called.
 28   #
 29   #   a = [ [:a,1,2], 2, :b, [:c,3], 9 ]
 30   #   a.to_h  #=> { [:a,1,2]=>2, :b=>[:c,3], 9=>nil } 
 31   #
 32   # Finally, a particular dispatch can be forced by 
 33   # specifying the +mode+ of conversion, eg. +:multi+,
 34   # +:splat+, +:flat+, +:assoc+, etc.
 35   #
 36   # Setting +mode+ to +true+ is the same as setting it +:multi+.
 37   # This has been left in for backward compatability.
 38   #
 39   # NOTE: The use of a +values+ parameter has been deprecated
 40   # because that functionality is as simple as:
 41   #
 42   #   array1.zip(array2).to_h
 43   #
 44   # CREDIT: Robert Klemme
 45   # CREDIT: Trans
 46   #
 47   #--
 48   # The +True+ option in the case statement provides some
 49   # backward compatability with the previous versions of this
 50   # method.
 51   #++

 53   def to_h(mode=nil)
 54     case mode
 55     when :splat
 56       return to_h_splat
 57     when :flat
 58       return to_h_flat
 59     when :multi, true
 60       return to_h_multi
 61     when :assoc

 94     pairs = true

104     end
